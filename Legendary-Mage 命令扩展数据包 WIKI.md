## 🎯 功能概述

Legendary-Mage 模组提供了一个强大的**游戏内命令系统**，允许玩家在不退出游戏的情况下，通过简单的命令为任何法术流派配置**元素标记兼容性**。

### ✨ 核心特性

| 特性           | 说明                  |
| ------------ | ------------------- |
| 🔥 **实时配置**  | 游戏内直接配置，无需编辑文件      |
| 🌐 **双语支持**  | 支持中文和英文命令           |
| ⚡ **热加载**    | 配置立即生效，无需重启         |
| 💾 **数据包导出** | 可导出为持久化数据包          |
| 🎨 **可视化**   | 卷轴 Tooltip 显示元素标记信息 |

***

## ⌨️ 命令系统

### 🇨🇳 中文版命令

```
/传奇法师 元素兼容性 [元素列表]
```

**子命令**:

- `/传奇法师 元素兼容性` - 查看当前手持卷轴的流派信息
- `/传奇法师 元素兼容性 <元素列表>` - 设置元素标记
- `/传奇法师 元素兼容性 info` - 查看详细信息
- `/传奇法师 元素兼容性 clear` - 清除配置
- `/传奇法师 元素兼容性 auto` - 自动检测元素
- `/传奇法师 元素兼容性 导出` - 导出为数据包

### 🇬🇧 英文版命令

```
/legendarymage elementcompat [element list]
```

**Subcommands**:

- `/legendarymage elementcompat` - View current spell school info
- `/legendarymage elementcompat <element list>` - Set element marks
- `/legendarymage elementcompat info` - View detailed info
- `/legendarymage elementcompat clear` - Clear configuration
- `/legendarymage elementcompat auto` - Auto-detect elements
- `/legendarymage elementcompat export` - Export to datapack

## 🚀 快速入门

### 第一步：手持卷轴

在主手拿着你想配置的法术卷轴或法术书。

### 第二步：配置元素

**示例 1**: 为咒刃流派配置血系和邪术元素

```bash
/传奇法师 元素兼容性 血 邪
```

**示例 2**: 为冰霜流派配置冰元素

```bash
/传奇法师 元素兼容性 冰
```

**示例 3**: 混合使用中英文

```bash
/传奇法师 元素兼容性 fire ice lightning
```

### 第三步：立即生效！

配置会**立即生效**，你现在使用该流派的法术就会造成配置的元素标记！

## 📦 数据包导出

### 为什么要导出？

- **持久化保存**: 游戏重启后配置不会丢失
- **分享配置**: 可以将配置分享给其他玩家
- **服务器使用**: 方便在服务器上部署

### 导出方法

#### 导出当前流派

手持卷轴，执行：

```bash
/传奇法师 元素兼容性 导出
```

#### 导出所有配置

空手执行：

```bash
/传奇法师 元素兼容性 导出
```

### 导出位置

```
config/legendarymage/datapacks/<namespace>/custom_schools/<school_id>.json
```

**示例**:

```
config/legendarymage/datapacks/irons_spellbooks/custom_schools/spellblade.json
```

### 导出文件格式

```json
{
  "name": "咒刃流派",
  "color": 7025413,
  "description": "Auto-generated element compatibility config for irons_spellbooks:spellblade",
  "target_school_id": "irons_spellbooks:spellblade",
  "compatible_elements": ["blood", "eldritch"],
  "element_mark_mapping": {
    "default": "blood",
    "condition_1": "eldritch"
  },
  "attribute_modifiers": {
    "spell_power_bonus": 0.0,
    "magic_resist_bonus": 0.0,
    "mana_cost_reduction": 0.0,
    "cast_time_reduction": 0.0
  }
}
```

### 使用导出的数据包

1. **复制到世界目录**
   ```
   复制 config/legendarymage/datapacks/<namespace>/
   到 saves/<你的世界>/datapacks/
   ```
2. **加载数据包**
   ```
   /reload
   ```
3. **完成！** 配置现在会在游戏启动时自动加载

## ⚡ 热加载机制

### 什么是热加载？

热加载允许你在**不重启游戏**的情况下，让配置立即生效。

### 工作原理

```
玩家输入命令
      ↓
命令系统更新配置
      ↓
热加载系统同步到缓存
      ↓
元素反应系统立即使用新配置
```

### 热加载 vs 数据包

| 特性   | 热加载     | 数据包        |
| ---- | ------- | ---------- |
| 生效速度 | 立即      | 需要 /reload |
| 持久化  | 否（重启丢失） | 是          |
| 适用场景 | 测试调试    | 正式使用       |
| 推荐用法 | 开发时使用   | 完成后导出      |

### 最佳实践

1. **开发/测试阶段**: 使用热加载快速调整
2. **确定配置后**: 导出为数据包
3. **正式游戏**: 使用数据包确保持久化

## ❓ 常见问题

### Q1: 为什么我的配置在重启后消失了？

**A**: 热加载的配置不会自动保存。你需要使用**导出功能**将配置保存为数据包。

```bash
/传奇法师 元素兼容性 导出
```

### Q2: 如何查看某个流派的当前配置？

**A**: 手持该流派的卷轴，执行：

```bash
/传奇法师 元素兼容性
```

### Q3: 可以同时配置多个元素吗？

**A**: 可以！用空格分隔多个元素：

```bash
/传奇法师 元素兼容性 火 冰 雷
```

### Q4: 配置会覆盖原有的数据包配置吗？

**A**: 不会。命令配置会**合并**到原有配置中，不会删除数据包的配置。

### Q5: 为什么 Tooltip 不显示元素标记？

**A**: 确保：

1. 你已经配置了元素
2. 你手持的是正确的卷轴
3. 如果问题持续，尝试重新登录或执行 `/reload`

### Q6: 英文版和中文版命令有什么区别？

**A**:

- 中文版只接受中文元素名
- 英文版只接受英文元素名
- 功能完全相同

### Q7: 如何删除某个特定元素？

**A**: 目前只能清除所有配置后重新设置：

```bash
/传奇法师 元素兼容性 clear
/传奇法师 元素兼容性 <你想保留的元素>
```

### Q8: 导出后如何修改已导出的数据包？

**A**: 你可以：

1. 直接编辑 JSON 文件
2. 在游戏中重新配置并再次导出（会覆盖原文件）

***

### 配置优先级

当同一流派有多个配置来源时，系统会**合并**所有配置：

1. **数据包配置** - 从 JSON 文件加载
2. **运行时配置** - 通过命令配置
3. **合并结果** = 数据包配置 ∪ 运行时配置

### 文件位置

| 类型      | 路径                                |
| ------- | --------------------------------- |
| 运行时配置缓存 | 内存中（重启丢失）                         |
| 导出数据包   | `config/legendarymage/datapacks/` |
| 世界数据包   | `saves/<世界>/datapacks/`           |
